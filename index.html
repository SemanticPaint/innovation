Dilemmas with innovation

How long can you operate at a sub-performing level while you do research and development?

- even while given time for think big thursdays, projects still march forward with deadlines

- lots of time spent in tooling; people take for granted the ease of publishing a modern-looking website

- R&D pace slow -- incorporate experiments with pilot projects.

How can you switch models when you're mired in legacy content and systems?

- this is why pilot projects are important -- don't make the wrong decision.

- difficulty of switching away from Drupal at Badgeville. already have a process, tooling, method.

- most tech writers are cogs in a wheel. decisioning power to make major tech changes limited.

What happens to your career skillset when you invest in an obscure technology?

- today the tools you know lands you the job; writing skills assumed

- companies want you to hit the ground running

- plug into company's existing toolset

How do you know if you're just wasting your time?

- best way to do something in new system may be unknown. lots of trial and error.

- spend a lot of time just figuring things out

XML and the web: Drifting apart?

- proliferation of REST APIs on web using JSON as default language format instead of XML

- reverberates through web stack toolchain

- not a lot of UX people want to work with XML; example from B. Colborn, from N. Kaplan, Badgeville setup

Technologies used by web developers

- XML not very commonly used in web publishing solutions

- convert XML to HTML, but hard to integrate into toolset even so

- Markdown is really popular and preferable

- Number of web development tools is exploding. This is where innovation is really hot.

Web tools to know

Bootstrap

- the bootstrap grid -- gets you responsive design easily

- add new components like magic with just styles

- lots of js fiddles showing how-tos, such as with collapsible section

jQuery

- jquery plugins for everything -- selecting, filtering, animating, interacting

- know simple way to integrate jquery and you expand your world significantly

- sidebar navigation, drop-downs, mini-TOCs, expand/collapse, filter

Markdown

- no tags to get in the way, so it's very readable

- where you fall short, use HTML. just be careful of spacing.

- a ton of web dev tools process Markdown.

3 Strategies for innovation

  1. Look outside the tech comm bubble.
  2. Leverage wisdom from the crowd.
  3. Accept that some efforts will fail.

- tech comm technologies are a speck in the landscape. tech comm is much larger than people in this space

- blog about what you do. really interesting to get feedback on ideas.

- even if you fail, there is serendipity. e.g., import DITA into WP failed, but learned how hard it is to use a tool against its intended design
